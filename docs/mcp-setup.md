# Connecting AI Tools to Manifold

Manifold exposes a single MCP endpoint your AI tools connect to. From the tool's perspective, it looks like one MCP server — but behind it sits every connector your organisation has configured, gated by your identity and the connectors you've enabled.

---

## Overview

Manifold acts as a single authenticated MCP server for your team. Your AI tool connects to one URL, signs in via OAuth, and gets access to every tool from every connector you've enabled — Pipedrive, Trello, Google Drive, and whatever else your admin has registered. You don't manage separate API keys in your AI tool. You manage your connectors once, in Manifold, and every tool that supports MCP picks them up automatically.

---

## What You'll Need

| Item | Where to get it |
|---|---|
| **MCP URL** | `https://your-domain.com/mcp` — your admin provides this, or you'll see it during onboarding |
| **Client ID** | Generated by your admin in Admin → Settings → OAuth Credentials |
| **Client Secret** | Generated at the same time — shown once, copy it immediately |
| **Authorization URL** | `https://your-domain.com/oauth/authorize` |
| **Token URL** | `https://your-domain.com/oauth/token` |

Your admin generates the OAuth client credentials. If you are the admin, go to **Admin → Settings → OAuth Credentials** and click **Generate credentials**. The Client Secret is shown once — copy it before closing the dialog.

---

## General Setup

These steps apply to any MCP-compatible AI tool.

1. Get the MCP URL from your admin. It will be `https://your-domain.com/mcp` where `your-domain.com` is your Manifold deployment.
2. Ask your admin to generate an OAuth client in **Admin → Settings → OAuth Credentials**. They'll give you a Client ID and Client Secret.
3. In your AI tool's settings, add a remote MCP server using the MCP URL.
4. Configure OAuth with these values:
   - **Authorization URL:** `https://your-domain.com/oauth/authorize`
   - **Token URL:** `https://your-domain.com/oauth/token`
   - **Client ID:** from your admin
   - **Client Secret:** from your admin
5. When prompted to authorise, sign in with your work Google account. You'll see a consent screen — click **Authorise**.
6. Your tools appear automatically. Only connectors you've enabled and configured show up.

## Tool-Specific Guides

### Claude (claude.ai / Claude for Teams)

Claude for Teams supports remote MCP servers at the organisation level. An admin configures the server once; all team members connect through it.

**As the Claude org admin:**

1. Go to [claude.ai](https://claude.ai) and open **Settings**.
2. Navigate to **Integrations** → **Remote MCP Servers**.
3. Click **Add server**.
4. Enter the **Server URL:** `https://your-domain.com/mcp`
5. Select **OAuth 2.0** as the authentication method.
6. Enter the **Client ID** and **Client Secret** generated in Manifold.
7. Set **Authorization URL** to `https://your-domain.com/oauth/authorize`.
8. Set **Token URL** to `https://your-domain.com/oauth/token`.
9. Save. Claude will prompt each user to authorise on first use.

**As a team member:**

When you open a new conversation in Claude, you'll see a prompt to authorise the Manifold MCP server. Click it, sign in with your work Google account, and you're done. Your tools are available immediately.

> UI paths in Claude may change as Anthropic updates the product. If these steps don't match what you see, look for **Integrations** or **MCP Servers** in your Claude org settings.

### Cursor

Cursor supports MCP servers in its settings panel, with OAuth configuration.

1. Open Cursor and go to **Settings** (⌘, on macOS or Ctrl+, on Windows/Linux).
2. Search for **MCP** or navigate to the **MCP** section.
3. Click **Add MCP Server**.
4. Enter a name (e.g. `Manifold`) and the **Server URL:** `https://your-domain.com/mcp`
5. Set the authentication type to **OAuth 2.0**.
6. Fill in:
   - **Client ID:** from your admin
   - **Client Secret:** from your admin
   - **Authorization URL:** `https://your-domain.com/oauth/authorize`
   - **Token URL:** `https://your-domain.com/oauth/token`
7. Save and restart Cursor if prompted.
8. On first use, Cursor will open a browser window for you to authorise. Sign in with your work Google account.

> Cursor's MCP configuration UI evolves frequently. If the MCP section isn't visible, check for an **Extensions** or **Tools** section, or consult the Cursor docs for the current path.

### Generic MCP Client (curl)

If you're building an MCP client or integrating programmatically, here's the full OAuth 2.1 PKCE flow manually.

**Step 1 — Generate PKCE values**

```bash
# Generate a code verifier (43–128 chars, URL-safe)
CODE_VERIFIER=$(openssl rand -base64 32 | tr -d '=+/' | cut -c1-43)

# Generate the code challenge: BASE64URL(SHA256(code_verifier))
CODE_CHALLENGE=$(echo -n "$CODE_VERIFIER" | openssl dgst -sha256 -binary | openssl base64 | tr -d '=' | tr '+/' '-_')

echo "Verifier: $CODE_VERIFIER"
echo "Challenge: $CODE_CHALLENGE"
```

**Step 2 — Open the authorization URL in a browser**

```
https://your-domain.com/oauth/authorize
  ?response_type=code
  &client_id=YOUR_CLIENT_ID
  &redirect_uri=YOUR_REDIRECT_URI
  &code_challenge=YOUR_CODE_CHALLENGE
  &code_challenge_method=S256
  &state=RANDOM_STATE_VALUE
```

Sign in with your Google account and authorise. You'll be redirected to your `redirect_uri` with a `code` query parameter.

**Step 3 — Exchange the code for a token**

```bash
curl -X POST https://your-domain.com/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=authorization_code" \
  -d "code=AUTHORIZATION_CODE_FROM_REDIRECT" \
  -d "redirect_uri=YOUR_REDIRECT_URI" \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET" \
  -d "code_verifier=YOUR_CODE_VERIFIER"
```

Response:

```json
{
  "access_token": "abc123...",
  "token_type": "Bearer",
  "expires_in": 7776000
}
```

**Step 4 — Call the MCP endpoint**

```bash
# List available tools
curl -X POST https://your-domain.com/mcp \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/list",
    "params": {}
  }'

# Call a tool
curl -X POST https://your-domain.com/mcp \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/call",
    "params": {
      "name": "pipedrive_list_deals",
      "arguments": { "status": "open" }
    }
  }'
```

## After Connecting

Once your AI tool is connected and authorised, calling `tools/list` returns only the tools from connectors you've enabled and configured. If you haven't enabled any connectors yet, the list will be empty.

Tools are scoped to you. Another user on the same Manifold instance sees their own tool list based on their own connector configuration.

---

## Enabling Connectors

Connectors don't appear as tools until you enable them.

1. Go to `https://your-domain.com/connectors`.
2. Toggle a connector on.
3. If the connector requires credentials (an API key, for example), a configuration panel slides in. Fill in the required fields.
4. Click **Test connection** to verify your credentials work.
5. Click **Save**. The connector's tools are now available in your AI tool.

Only connectors that are both **enabled** and **configured** appear in your tool list. A connector that's enabled but has no credentials saved will not contribute tools.

---

## Token Expiry

Bearer tokens last 90 days. When a token expires, your AI tool will prompt you to re-authorise — this typically looks like a reconnect or sign-in prompt.

There is no silent refresh in v1. When prompted, sign in with your Google account again. The process takes under a minute.

---

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| No tools appear | No connectors enabled, or all enabled connectors are deprecated | Go to `/connectors` and enable at least one connector |
| "Connector is not configured" error | Connector is enabled but credentials haven't been saved | Go to `/connectors`, open the connector, and save your credentials |
| 401 Unauthorized | Token expired or revoked | Re-authorise in your AI tool |
| A connector's tool call fails | The connector's MCP server is unreachable or returned an error | Contact your admin — the connector endpoint may be down |
| "Unknown tool" error | A connector was deprecated after the tool list was cached | Call `tools/list` again to get the current tool list |
